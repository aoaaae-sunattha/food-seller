3 Approaches to Build This System

All use Next.js + Google Sheets. The difference is scope and complexity.

A

⭐ Smart Receipt App + Simple Forms Recommended

Next.js web app (tablet-optimized, Thai UI). 4 screens: Upload Receipt, Deduct Stock, Record Daily Sales, Settlement. Receipt photo → AI (Claude Vision) extracts items + prices → user confirms → saved to Google Sheets. Manual forms for stock deduction and end-of-day. Sheets has tabs: Purchases, Stock, Daily Sales.

Pros

  • Build in 2–3 weeks
  • AI receipt scanning saves time
  • She can edit Sheets directly if needed
  • Low maintenance

Cons

  • No automatic cost-per-dish calculation (manual estimate)
  • No real-time stock alerts
B

Mini POS with Recipe Engine

Everything in option A, plus: define each dish as a recipe (e.g. Pad Thai = 200g noodles + 2 eggs + …). When she records how many boxes sold, the system auto-deducts ingredients from stock and calculates exact cost per box. More structured data.

Pros

  • Accurate cost-per-dish
  • Auto stock deduction from sales
  • Real profitability tracking

Cons

  • 4–6 weeks to build
  • Recipes must be set up correctly upfront
  • More complex to maintain
C

Google Sheets + Apps Script Only

No custom web app. A Google Apps Script handles receipt uploads via a Google Form. All logic (stock updates, expense tracking) runs inside Google's ecosystem. Share the spreadsheet with her accountant directly.

Pros

  • Zero hosting cost
  • Accountant-friendly
  • No deployment needed

Cons

  • No AI receipt scanning
  • Clunky on tablet
  • Hard to extend later

My recommendation: Start with Option A. It solves 90% of her pain immediately. If she needs recipe-based cost tracking later, that's an upgrade path — not a rebuild.
