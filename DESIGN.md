## Professional Modern Thai Design System (Kitchen OS Edition)

This document serves as the **Visual Source of Truth** for the Lawan Thai Traiteur application. All UI components, layouts, and interactions generated by AI agents MUST strictly adhere to these specifications to ensure a premium, commercial-grade user experience optimized for both **Web (Desktop)** and **iPad (Tablet)** users.

---

### 1. Visual Mood & Brand Identity
- **Vibe:** Sophisticated, Calm, Trustworthy, Data-Driven "Kitchen OS".
- **Target Devices:** Web (Desktop) & iPad / Tablet (Dual-optimized).
- **No Emojis:** Use professional SVG icons (Lucide-React or Heroicons) for all status indicators, buttons, and navigation.
- **Aesthetic:** "Siam Minimalist" â€” clean white spaces, crisp typography, subtle card elevations, and high-contrast semantic states for quick operational reading.

### 3. Typography (Enhanced Readability)
- **Primary Font:** Inter / System UI Sans (Clean, high-readability for operational data).
- **Thai Font:** Noto Sans Thai (Weights: 400, 500, 600).
- **Scale (Responsive - Optimized for Visibility):**
  - `Data Value`: 40px / 2.5rem (Bold, Slate Deep) - Primary metric numbers.
  - `Header 1`: 36px / 2.25rem (Desktop), 32px / 2rem (Tablet) - Page titles.
  - `Header 2`: 24px / 1.5rem (Semi-bold) - Section/Card titles.
  - `Body`: 20px / 1.25rem (New iPad Standard) - Main operational text.
  - `Caption`: 16px / 1rem (Metadata, low emphasis, timestamps).

---

### 4. Layout Architecture (Bento Grid OS)
#### A. Navigation (Sidebar)
- **Desktop (Web):** Fixed **Left Sidebar** with full labels (width `240px`).
- **Tablet (iPad):** Collapsible or **Icon-only Sidebar** (width `80px`) to maximize content space.
- **Active State:** Highlighted with a `Cinnabar Orange` left border and text/icon color.

#### B. Dashboard Structure
Use a high-density "Bento Grid" layout to display operational status at a glance.
- **Top Row (Metrics):** 3-4 small cards showing key numbers (e.g., `Today's Sales`, `Pending Receipts (OCR)`, `Low Stock Items`). Layout: Icon + Title at top, large `Data Value` number below.
- **Middle Row (Overview):** Large spans for charts (e.g., Weekly Sales Bar Chart) alongside a secondary data panel (e.g., Monthly Summary Stats).
- **Bottom Row (Activity Feed):** List views for "Recent Purchases" or "Stock Movements".

#### C. Touch & Click Targets
- **Interactive Elements:** Minimum hit area of **44x44px** (mandatory for iPad, beneficial for desktop).
- **Gaps & Margins:**
  - `Desktop`: Page margin `40px`, Grid gap `24px`.
  - `Tablet`: Page margin `24px`, Grid gap `16px`.

---

### 5. Component Specifications (Commercial Grade)
#### A. Cards & Surfaces
- **Background:** `Surface White` (`#FFFFFF`).
- **Border:** `1px solid var(--mist-gray)` (`#F3F4F6`).
- **Shadow:** Soft elevation (e.g., `0 2px 8px -2px rgba(0,0,0,0.05)`).
- **Border Radius:** `16px` for main cards, `8px` for inner elements/buttons.

#### B. Status Badges (Pills)
Used extensively in activity feeds (e.g., OCR Status).
- **Completed / Success:** `Emerald Green` background with 15% opacity, text in solid `Emerald Green`.
- **Pending / Processing:** `Amber Gold` background with 15% opacity, text in solid `Amber Gold`.
- **Alert / Low Stock:** `Error Red` background with 15% opacity, text in solid `Error Red`.
- **Shape:** Fully rounded (pill shape, `9999px` border-radius).

#### C. Activity Feed Lists
- **Structure:** Left: Circular icon or thumbnail (e.g., Receipt icon or Ingredient image). Middle: Title (e.g., "Purchased Chicken") and subtitle (e.g., "ŕ¸ż 1,200 â€˘ 10 mins ago"). Right: Status Badge.
- **Separation:** Use a `1px solid var(--mist-gray)` bottom border between list items.

#### D. Buttons & Inputs
- **Primary Button:** `Cinnabar Orange` background, `White` text.
- **Secondary Button:** `Mist Gray` background, `Slate Deep` text.
- **Data Inputs:** Underlined or subtle border that turns `Cinnabar Orange` on focus.
- **Active (iPad):** Scale down 5% to provide tactile feedback on touch.

---

### 6. Iconography Strategy (SVG Only)
- **Library:** Lucide-React (Preferred).
- **Stroke Width:** `1.5px` to `2px` (Balanced for high-DPI screens).
- **Icons Mapping (Project Specific):**
  - Dashboard: `LayoutDashboard`
  - Receipts/OCR: `ScanLine` or `FileText`
  - Stock: `Package` or `Boxes`
  - Sales: `CircleDollarSign` or `TrendingUp`
  - Config/Menus: `Settings` or `ChefHat`

---

### 7. UX Anti-Patterns (Avoid)
- **NO NON-EXISTENT FEATURES:** Do not design for "Tables," "Wait Times," or "Orders in Progress." Stick strictly to Sales, Purchases, Stock, and OCR.
- **NO EMOJIS:** Absolutely no `âś…`, `âťŚ`, `đź“¦`. Use themed SVG icons.
- **NO HOVER-ONLY INFO:** Crucial information must be visible without a hover state, as iPad users won't have one.
- **NO FIXED HEIGHTS:** Allow cards and containers to grow with content.

---

### 8. Implementation Note
When generating React components, utilize Tailwind's responsive prefixes (`md:`, `lg:`) to switch layouts between Web and iPad. Always include `className="touch-manipulation"` on interactive parent elements for optimized iPad responsiveness.
